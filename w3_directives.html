AngularJS lest you extend HTML with new attributes called directives
AngularJS has a set of built -in directivs which offers functionality to your applications
AngularJS aslo lets you define your own directives.

ng-*

AngularJS directives are extended HTML attributes with the prefix ng-
  • ng-app - initialized the application. also tells AngularJS that the div elment is the owener of the application
  • ng-init - initialized the application data
  • ng-model - binds the value of HTML controles to the application data
    • Provide type validation for application data (number, email, required)
    • Provide status for application data (invalid, dirty, touched, error)
    • Provide CSS classes for HTML elements
    • Bind HTML elments to HTML forms
  • ng-repeat - repeats an HTML element

Data Binding

ng-init will bind a value to the HTML using ng-model
ng-init is not very commonly used.

Repeating HTML Elements

  • {{ x }}
  • {{x.name + ', ' + x.country}}

New Directives

New directives are created by using .directive function
You can invoke a directive by using
  • Element Name - <w3-test-directive> </w3-test-directive>
  • Attribute - <div w3-test-directive> </div>
  • Class - <div class="w3-test-directive"> </div>
  • Comment - <!-- directive: w3-test-directive -->

Restictions

You can restrict your directives to only be invoked ny some fo the methods
The legal restrict values are:
  • E for Element name
  • A for Attribute
  • C for Class
  • M for Comment
